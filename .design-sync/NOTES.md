# design-sync NOTES (prism.oshi.tw)

This repo is a **Next.js 16 app**, not a standalone component library — it's
synced to Claude Design as a design system. That drives most of the config
shape below. Read this before any re-sync.

## Why the config looks the way it does
- **No `dist/` → synth entry.** There is no built component library. `cfg.entry`
  points at `.design-sync/.cache/ds-entry.mjs`, generated by
  `.design-sync/gen-entry.mjs` (run inside `cfg.buildCmd`). Every component is a
  **default export**, so the entry uses `export { default as <Name> }` — the
  converter's stock `export * from` would drop them. `gen-entry.mjs` also emits
  `_ds-shim.mjs` (imported first) defining a minimal `globalThis.process` so the
  bundled **Next.js runtime** (`next/link`, `next/navigation` pull in
  `process.env.__NEXT_*` reads) doesn't throw "process is not defined".
- **`cfg.buildCmd` MUST run before the converter** every build/re-sync: it
  regenerates the entry+shim AND compiles Tailwind (`app/globals.css` +
  `@tailwind` directives) into `.design-sync/.cache/compiled.css`, which is
  `cfg.cssEntry`. The raw `globals.css` is not usable (uncompiled directives).
- **`cfg.tsconfig` → `.design-sync/tsconfig.sync.json`** (NOT the app's
  tsconfig). It remaps `@/*` to the repo root AND stubs `next/link` +
  `next/navigation` (`.design-sync/stubs/`) so components render outside Next's
  app router. The app's own build is unaffected.
- **Preview harness / provider.** `cfg.provider = PreviewProvider`, defined in
  `.design-sync/preview-harness.tsx` and merged via `cfg.extraEntries`. It mounts
  the app's REAL context providers (Streamer/Player/Playlist/Liked/RecentlyPlayed)
  seeded with mock data (a current track, 5-song queue, 2 playlists, 4 liked, 4
  recent). A no-op `window.YT` stub is installed so `PlayerProvider` never loads
  the real IFrame API. Because extraEntries + the main entry bundle in ONE esbuild
  pass, the provider and the components share the same context module instances.
- **`.design-sync` is in the app's `tsconfig.json` `exclude`** so the committed
  `.tsx` sync files (harness, stubs, previews) never enter the app's
  `next build` typecheck.
- **`guidelinesGlob: []`** — the repo's `docs/*.md` (vod-export guides) are ops
  docs, not design guidelines; shipping them would mislead the design agent.
- **Overlay components** (`NowPlayingModal`, `*Panel`, `CreatePlaylistDialog`,
  `BottomSheet`, `MiniPlayer`, `AddToPlaylistDropdown`) portal to `document.body`
  and are fixed-position → each has `cfg.overrides.<Name>.cardMode: "single"` +
  a viewport. `AlbumArt` uses `cardMode: "column"` (its size-sweep row is wide).

## Known render warnings (triaged as legitimate — not new issues)
- `[FONT_REMOTE] "DM Sans"` — `globals.css` `@import`s DM Sans from Google
  Fonts; it loads at runtime. Expected; not shipped as a local font.
- `[RENDER_THIN] DiscordIcon` — a monochrome icon renders no text, so the thin
  heuristic fires even WITH the authored preview. Benign: the icons render
  correctly (confirmed visually). Leave it.

## Environment gotchas
- The Bash working directory PERSISTS across calls. A stray `cd ds-bundle` (to
  list files) once broke a later build that used repo-relative paths. Always run
  converter commands from the repo root; `cd` back if you moved.
- esbuild's postinstall is blocked by the user's npm `allow-scripts`, but the
  `@esbuild/darwin-arm64` optional dep provides the binary — esbuild works.
- Playwright's chromium lives at macOS `~/Library/Caches/ms-playwright/`
  (`chromium-1223`), not `~/.cache/`.

## Re-sync risks / watch-list
- **Mock data in `preview-harness.tsx` tracks component APIs.** If a context's
  hook shape or a component's props change upstream, the harness seeding or a
  preview's mock props may silently go stale (component renders empty, not an
  error). Re-grade after any context/component refactor.
- **The `next/*` stubs** (`.design-sync/stubs/`) implement only the surface the
  components use today (`usePathname`, default `Link`). A component adopting a new
  `next/navigation` export needs the stub extended.
- **`PreviewProvider` is exposed on the global** (via extraEntries) and documented
  in `conventions.md` as the demo wrapper. It carries fixed sample data by design.
- **Component groups**: all 25 land in `general/`. If a real grouping is wanted,
  add `docsMap` category stubs.

## Preview authoring learnings (fan-out)
- **The per-card render background is flat white** (`emit.mjs`), so the app's
  light-theme translucent-white surface tokens (`--bg-surface-muted` `#FFFFFF80`,
  `--border-glass`, `--bg-surface-frosted`) vanish on a bare card. Controls
  (ThemeToggle, ProgressBar track, VolumeControl tail) are staged on tokened
  panels (page gradient / accent gradient) INSIDE their previews.
- **Previews may only use Tailwind classes already present under `app/`** — the
  content scan excludes `.design-sync/previews`, so novel utility classes don't
  compile. Use inline `style={{…}}` (with `var(--*)`) otherwise.
- **`BottomSheet` is viewport-width-gated** (`lg` = 1024px): panels captured
  below 1024 → the mobile bottom-sheet variant (the more distinctive view);
  ≥1024 → desktop right side-panel. Its surface is hard-coded dark, so `children`
  must be light-on-dark (never `--text-primary`, which is dark in light theme).
- **`SidebarNav` + `TimelineRow` use in-preview scoped CSS shims** to force their
  `lg:`-gated desktop layout at the 900px capture width. Cleaner alternative if
  ever reworked: give each a `viewport ≥ 1024` override and delete the shim.
- **`MiniPlayer`** (`position: fixed; bottom: 0`, no in-flow height) is wrapped in
  a `transform: translateZ(0)` fixed-height stage in its preview so the bar docks
  inside the card instead of escaping it.
- **`AddToPlaylistDropdown`** opens its playlist menu via internal state on click;
  with no `open`/`defaultOpen` prop it can't render open statically — graded on
  its (styled) trigger.
- **`Toast`** has a 2000ms auto-hide timer; captures fine before it fades.
- **Grid-overflow card modes** (from validate): `ProgressBar`, `NowPlayingControls`,
  `MobileSearchRow`, `TimelineRow` → `cardMode: column`; `Toast` → `single`
  (fixed/portal). Panels + modal → `single` with tuned viewports.
